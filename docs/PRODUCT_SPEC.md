# 공간 배치 스튜디오 — 제품 기획서 (Product Spec)

> 이 문서는 프로젝트의 전체 비전과 요구사항을 담는 살아있는 기획서입니다. 단계가 진행될 때마다 갱신합니다.

## 1. 비전

건축 전문가가 아닌 숙박업·카페·상가·공간 창업자가 도면, 주소, 제품 링크(또는 제품명)만 입력해도 임시 공간 시안을 쉽게 만들 수 있는 가상 건축·공간 배치 디자인 앱.

핵심 원칙: 전문가용 CAD가 아니라 **초보자도 클릭 몇 번으로 결과를 볼 수 있는 앱**.

## 2. 현재 구현 상태 — Phase 1 (완료)

- 건물 생성 폼(대지면적/건폐율/용적률/높이/층수) → 풋프린트 및 층 자동 생성
- 층 패널: 층 클릭으로 전환, **층별 오브젝트는 완전히 분리 저장**(구현·검증 완료 — 아래 §3 참고)
- 2D 캔버스(react-konva): 풋프린트 외곽선 + 배치된 오브젝트 렌더링
- 팔레트: 기본 시설 10종(계단/엘리베이터/기계실/물탱크실/전기실/EPS실/소방/관리실/공용복도/설비공간) + 일반 오브젝트, 클릭으로 활성 층에 배치
- 오브젝트 조작: 드래그 이동, 리사이즈/회전(Transformer), 좌우/상하 반전, 복사, 삭제, 층 이동, 그룹/그룹해제(그룹째 드래그 이동)
- zustand + localStorage로 자동 저장/복원
- 디자인/제품 탭 골격 (제품 탭은 placeholder)

미포함(의도적으로 다음 단계로 미룸): 도면 업로드/분석, 주소 지오코딩, 제품 링크 스크래핑, AI 이미지 생성, 3D 렌더링, mm 단위 정밀 입력.

## 3. Phase 2 요구사항 (사용자 확정 — 반드시 반영)

### 3.1 건물 생성 시작 화면
기존 숫자 입력 폼에 아래 입력 영역이 **반드시 추가**되어야 함:
- 도면 이미지 파일 업로드
- 도면 PDF 파일 업로드
- 주소 입력창
- Google API Key 입력창

업로드된 도면 이미지/PDF는 앱이 자동 분석해 대지 형태·건물 구조를 최대한 유사하게 생성. 인식이 부정확하거나 일부만 인식되면 **사용자가 직접 수정**할 수 있어야 함(캔버스에서 풋프린트/치수를 편집 가능하게).

### 3.2 주소 기반 입지 생성
Google API Key 입력만으로는 부족하고, **주소 입력창이 별도로 반드시 존재**해야 함.
동작 순서: **주소 입력 → Geocoding으로 위치 분석 → 대지 생성 → 건물 생성**.
Google Maps/Geocoding API를 사용해 해당 주소의 위치와 주변 입지를 최대한 유사하게 반영.

### 3.3 층별 저장 기능 — ✅ Phase 1에서 이미 구현/검증됨
- 각 층은 독립적으로 저장되며, 층 이동 후에도 이전 층의 배치가 그대로 유지됨 (예: 1F는 리셉션/카페 테이블/의자/계단, 2F는 객실/침대/복도/창고, 3F는 라운지/테이블/소파처럼 층별로 완전히 다른 구성 가능).
- `objectsSlice`의 모든 오브젝트가 `floorId`로 층에 귀속되고, 캔버스는 `activeFloorId`로 필터링해서 렌더링 — 층 전환 시 데이터 손실 없음. Playwright로 층 전환 + 새로고침 시나리오 모두 검증 완료.
- 추가 작업 불필요, 문서화 목적으로만 명시.

### 3.4 미니어처 크기 직접 입력 기능 (신규)
캔버스의 오브젝트를 **더블클릭하면 상세 설정 모달**이 열려야 함. 모달에서 직접 입력 가능한 값:
- 가로(mm), 세로(mm), 높이(mm)
- 회전 각도
- 이름
- 메모

가로/세로(mm)를 입력하면 캔버스 위 미니어처 크기가 실제 비율에 맞게 즉시 반영되어야 함 (예: 1200mm × 700mm 입력 → 그 비율의 크기로 자동 조정).

### 3.5 제품 추가 입력창 (신규 — 2026-07-07 수정: 링크 파싱 대신 사진 업로드로 확정)
전용 "제품 추가" 영역 필요, 두 가지 생성 방식:
1. **제품 사진 입력**: 쿠팡/네이버/아마존 등 제품 링크를 파싱하는 대신, 사용자가 가진 사진을 직접 등록.
   - 파일 업로드(클릭해서 선택)
   - 클립보드 붙여넣기(Ctrl+V로 복사한 이미지 바로 붙여넣기)
   - 드래그 앤 드롭(사진 파일을 입력창에 끌어다 놓기)
   - 위 세 가지 중 어떤 방식으로 넣든 앱이 사진을 분석해 배경을 제거하고, **그 사진 그대로**(AI로 다시 그리지 않고) 미니어처 오브젝트로 변환. 즉 사용자가 가진 실제 제품 사진을 그대로 배치용 오브젝트로 씀.
2. **텍스트 입력**: "테이블", "의자", "침대", "카페 카운터", "리셉션 데스크" 등 제품명 입력 → AI가 미니어처 이미지 자동 생성 (사진이 없을 때만 사용하는 경로)

필요 UI 요소: 제품 사진 업로드/붙여넣기/드래그 영역 / 제품명 텍스트 입력창 / 생성 버튼 / 생성된 제품 목록 / 제품 미리보기 / 캔버스에 추가 버튼 / 드래그 앤 드롭 배치.

> ~~제품 링크 입력(쿠팡/네이버쇼핑/아마존 URL 파싱)~~ 은 CORS·ToS 리스크가 있어 제외하고, 사용자가 직접 사진을 제공하는 방식으로 대체함(§4 리스크 항목도 이에 맞춰 갱신).

### 3.6 제품 생성 후 배치 기능 (신규)
생성된 제품은 카드 형태로 목록에 저장되고, 클릭 또는 드래그로 원하는 층의 원하는 위치에 배치 가능. 배치 후 지원되어야 하는 조작:
드래그 이동 · 크기 조절 · 더블클릭 mm 직접 입력 · 회전 · 좌우 반전 · 상하 반전 · 삭제 · 복사 · 다른 층으로 이동 · **이름 변경(신규)**.

### 3.7 핵심 사용 흐름 (반드시 지켜야 할 UX 순서)
1. 도면 이미지 또는 PDF 업로드 (또는 건너뛰고 주소 입력)
2. 주소 입력
3. Google API Key 입력
4. 층수 입력
5. 자동 건물 생성
6. 층 선택
7. 제품 사진 업로드/붙여넣기/드래그 또는 제품명 입력
8. 미니어처 자동 생성
9. 캔버스에 배치
10. 더블클릭해서 mm 단위 크기 조정
11. 층별로 저장 (자동, 이미 구현됨)
12. 다시 열어도 기존 작업 유지 (자동, 이미 구현됨)

## 4. 구현 전 반드시 짚어야 할 아키텍처/리스크

이 항목들은 요구사항을 축소하려는 것이 아니라, 그대로 구현했을 때 실제로 막히는 지점을 미리 표시해두는 것입니다.

- **Google API Key 처리 — 확정: Cloudflare Worker 프록시**. 사용자가 입력한 키를 프론트엔드에서 그대로 쓰면 브라우저 개발자도구로 노출되므로, Cloudflare Worker를 백엔드 프록시로 두고 키는 Worker 환경변수(시크릿)에만 보관. 클라이언트는 Worker의 자체 엔드포인트를 호출하고, Worker가 Google Maps/Gemini API를 대신 호출해서 결과만 돌려줌. 이번 Phase 2 작업에 Cloudflare Worker 프로젝트 신설이 포함됨.
- **도면 이미지/PDF 분석**: Gemini(멀티모달)로 이미지에서 대략적인 외곽선·치수를 추출하는 것은 가능하지만 100% 정확하지 않음 — 사용자가 이미 "부정확하면 직접 수정" 요구사항을 명시했으므로, 분석 결과는 항상 "제안값"으로 캔버스에 얹고 사용자가 조정하는 흐름으로 설계.
- **제품 이미지 처리 — 확정: 링크 스크래핑 대신 사용자 사진 업로드/붙여넣기/드래그**. 쿠팡/네이버/아마존 링크를 서버에서 파싱하는 방식은 CORS·ToS 리스크가 있어 채택하지 않고, 사용자가 가진 사진을 업로드·클립보드 붙여넣기·드래그로 직접 등록 → 배경 제거만 거쳐 그대로 미니어처로 사용. 배경 제거는 브라우저에서 무료로 고품질로 하기 어려워 Cloudflare Worker에서 처리(무료/오픈소스 배경 제거 모델 또는 API 연동)하는 쪽으로 진행.

## 5. Phase 2 실행 순서 (제안, 확정 아님)

1. mm 단위 상세 설정 모달 (§3.4) — 외부 API 불필요, 바로 구현 가능
2. Cloudflare Worker 프록시 골격 구축 (Google API Key/Gemini 호출을 서버 쪽으로 이전)
3. 제품 추가 패널: 사진 업로드/붙여넣기/드래그 → 배경 제거(Worker 경유) → 미니어처 등록, + "텍스트 입력 → AI 생성" 경로
4. 주소 입력 → Geocoding 입지 생성 (§3.2) — Google Maps Geocoding (Worker 경유)
5. 도면 이미지/PDF 업로드 분석 (§3.1) — Gemini 비전 (Worker 경유)
5. 제품 링크 파싱 (§3.5) — 네이버 공식 API 우선, 나머지는 폴백
